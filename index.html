A2 Game Engine

A custom 2D game engine built from scratch in C++ with Lua scripting, SDL2 rendering,
Box2D physics, and Xbox controller support with live button remapping.
Built by Joel Martinez — University of Michigan EECS 498.

Controller Support

Full Xbox controller support via SDL2 GameController API. Buttons, analog sticks, and triggers are all exposed to Lua scripts via LuaBridge bindings. Players can remap any action to any button through an in-game pause menu — no engine restart required.

Lua Scripting

Game logic is fully scriptable in Lua via LuaBridge. Every actor in the scene can have components — Lua tables that implement OnStart, OnUpdate, and OnDestroy lifecycle hooks. The engine exposes Actor, Input, Audio, Image, Camera, Controller, and Scene namespaces to Lua.

Box2D Physics

Integrated Box2D middleware for rigid body physics, collision detection, and raycasting. Physics components are created and managed through Lua, with an EventBus system for collision callbacks and deferred component removal to avoid mid-update invalidation.

Technologies

Built over 10 weeks of EECS 498 — Game Engine Architecture. Every system from rendering to scripting to physics was implemented from scratch on top of industry-standard libraries.

C++

Core engine written in modern C++17. Actor/component architecture, deferred destruction, template caching, and spatial hashing for large scenes.

Lua + LuaBridge

Game logic fully scriptable in Lua. LuaBridge exposes C++ functions and classes to Lua with automatic type conversion and inheritance support.

SDL2

Windowing, rendering, audio, input, and controller support. SDL_GameController API powers the Xbox controller feature with hot-plug detection.

Box2D

Rigid body physics, collision detection, raycasting, and contact callbacks. Integrated via an EventBus system with deferred component removal.

GLM

Math library for vectors and transforms. glm::vec2 exposed directly to Lua for position and force calculations.

RapidJSON

Fast JSON parsing for scene files, actor templates, and game config. Template document caching reduces repeated disk reads.
